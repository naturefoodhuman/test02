# 技能：数据源管理与数据质量（工厂通用能力 FB-5）

> 老板要点2：数据源和数据质量非常重要，务必重视。
> 分工：**老板出定性判断（来源分级/黑名单），系统出量化（权重/交叉验证/忠实度）。**

## 核心机制（业界公认做法）
1. **来源分级 source tiering**：每个来源归一个可信度等级 → 系统映射为权重。
   - 官方/法院/法规 = 1.0；律所实务/专业机构 = 0.8；主流媒体 = 0.5；自媒体/论坛 = 0.3；黑名单 = 0.0。
2. **多源交叉验证**：关键结论需 ≥2 个独立可信源印证；单源孤证降权并标注。
3. **Faithfulness 忠实度校验**：生成内容必须能从来源推导（用 GLM 当裁判），防法律幻觉。
4. **Reranker 重排**：检索后按相关性+质量重排，过滤噪声。
5. **去软文/去私货**：识别引流、情绪煽动、夸大特征 → 降权。

## sources.yaml 工作流（关键）
- **系统先充分调研，产出 `sources.yaml` 初版**（带每个源的等级 + 理由 + 官方网址）。
- **老板后续微调共同维护**：调等级、加黑名单（如"某博主不可信"）。老板只给定性，权重由等级映射自动算。
- 模板见各项目的 `sources.yaml`（首例：projects/debt-collection/sources.yaml）。

## 检查清单
- [ ] 有 sources.yaml 且每个源标了等级+理由
- [ ] 关键结论有多源交叉验证
- [ ] 生成内容过了忠实度校验（无编造法条/数据）
- [ ] 黑名单来源已排除

## 反模式
- 不分来源一锅炖（自媒体软文和官方法规等权重）。
- 单一来源就下结论。
- 让老板去算权重（他只该给定性判断）。

## 输出物
- 项目级 `sources.yaml`（可维护）+ 带可信度标注的数据。
