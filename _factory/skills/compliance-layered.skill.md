## description
实现分层决策架构：铁闸规则（硬编码）→ AI 参考判断 → AI 生成建议。确保合规审查不会因 AI 幻觉而漏判/误判。

## knowledge

### 第一层：铁闸规则（硬编码，不依赖 AI）
这些是绝对红线，无论 AI 如何判断，铁闸规则优先：

| 铁闸编号 | 规则 | 违规示例 | 处罚依据 |
|---------|------|---------|---------|
| GATE-01 | 绝不建议暴力/威胁/恐吓 | "去威胁他" | 《刑法》第293条 |
| GATE-02 | 绝不建议侵犯隐私 | "爆通讯录" | 《民法典》第1032条 |
| GATE-03 | 绝不建议伪造证据 | "假借条" | 《刑法》第307条 |
| GATE-04 | 绝不建议冒充公职 | "装法院的人" | 《刑法》第279条 |
| GATE-05 | 绝不建议非法限制人身自由 | "关起来" | 《刑法》第238条 |
| GATE-06 | 绝不建议骚扰无关第三人 | "找他家人要钱" | 《治安管理处罚法》 |
| GATE-07 | 利率不得超过 LPR 4 倍 | "月息 5%" | 《民间借贷司法解释》第26条 |

**执行逻辑**：
```
if 文本匹配任何铁闸规则的关键违规词（非否定语境）:
    return BLOCKED + 铁闸违规原因
else:
    进入第二层
```

### 第二层：AI 参考判断（LLM 辅助分析）
铁闸通过后，由 AI 进行以下判断：
- 是否属于灰色地带（需加合规限定）
- 是否存在隐性违规（如心理操控、软暴力暗示）
- 措辞是否有歧义（可能被误解为违规建议）

**判断维度**：
1. 合法性：建议是否有法律依据
2. 合规性：建议是否符合行业规范
3. 适当性：建议的措辞是否得当
4. 完整性：是否遗漏了重要合规提示

### 第三层：AI 生成建议（输出最终合规建议）
综合铁闸和 AI 参考判断，生成结构化建议：

```
【合规审查结果】
- 铁闸检查：✅ 通过 / ❌ 未通过（原因：XXX）
- AI 参考判断：
  - 合法性：✅ / ⚠️ / ❌
  - 合规性：✅ / ⚠️ / ❌
  - 适当性：✅ / ⚠️ / ❌
  - 完整性：✅ / ⚠️ / ❌
- 最终建议：[具体修改意见]
- 免责声明：本报告为辅助参考，重大决策请咨询执业律师。
```

### 否定语境识别规则
关键词前出现否定词 → 不属于违规：
- 否定词列表：["绝不", "不得", "禁止", "严禁", "不要", "不能", "切勿", "避免", "不建议", "反对"]
- 示例："绝不可威胁恐吓" → ✅ 合法（否定语境）
- 示例："建议威胁恐吓" → ❌ 违规（肯定语境）
