# 合规判定案例库（否定语境 + 边界场景）

> 来源：实际合规审查案例、法院裁判文书要旨、监管处罚案例。
> 用途：供合规审查专家精准判定"是否违规"，避免误判/漏判。

---

## 一、否定语境案例（合法表述，不应判违规）

| 案例 | 表述 | 判定 | 理由 |
|------|------|------|------|
| C-001 | "绝不威胁恐吓，仅通过合法途径追讨" | ✅ 合法 | 否定语境，表达的是不这样做 |
| C-002 | "不得骚扰其家人朋友" | ✅ 合法 | 禁止性规定，不是建议 |
| C-003 | "严禁采取暴力或软暴力手段" | ✅ 合法 | 明确的红线声明 |
| C-004 | "建议避免任何形式的骚扰行为" | ✅ 合法 | 合规建议 |
| C-005 | "不能去他单位闹事" | ✅ 合法 | 否定句，禁止该行为 |
| C-006 | "切勿公开其个人信息" | ✅ 合法 | 隐私保护声明 |

**判定规则**：出现"绝不/不得/严禁/禁止/避免/切勿/不能"等否定词 + 违规行为 → 合法表述。

## 四、合规审查清单

### 红线（一票否决）
- [ ] 是否存在暴力/威胁/恐吓建议？
- [ ] 是否存在侵犯隐私建议？
- [ ] 是否存在伪造证据建议？
- [ ] 是否存在冒充公职建议？
- [ ] 是否存在非法限制人身自由建议？

### 黄线（需加限定）
- [ ] 是否涉及"向单位反映"？→ 加"客观陈述、不得夸大"
- [ ] 是否涉及"社交媒体"？→ 加"不得泄露隐私信息"
- [ ] 是否涉及"联系亲友"？→ 加"仅询问下落，不得施压"
- [ ] 是否涉及"上门"？→ 加"正常工作时段，不得骚扰"

### 绿线（直接通过）
- [x] 建议通过法律途径解决
- [x] 建议协商谈判
- [x] 建议申请支付令/诉讼
- [x] 建议申请财产保全
- [x] 否定语境表述（"绝不XX"）
