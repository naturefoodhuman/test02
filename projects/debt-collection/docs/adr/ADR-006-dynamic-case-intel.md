# ADR-006：动态案件博弈（情报库 + 策略随情报重算）
- 状态：已接受
- 日期：2026-06-12
- 背景：老板第16轮指出——讨债是持续动态博弈，新情报(如债务人当选村支书、微信沟通)随时进来，需像真正的债务律师持续追踪、更新法务知识、重评策略；不能"一次性出报告就完事"。
- 决策：V1 引入「案件时间线 + 情报库 + 策略动态重算」：
  - intel 模块记录每条情报(来源/可信度/策略影响)；
  - timeline 把债务关键日期+沟通+情报+取数聚合成时间线；
  - strategy 在新情报后做"增量策略更新"并识别合法博弈筹码；
  - knowledge 模块持续从法规库取法条+本地执行口径(防幻觉)。
- 合规护栏：所有"施压"必须合法(依法向任职单位反映真实欠债=正当维权；禁捏造/威胁/骚扰无关第三人)，由 compliance 把关。
- 备选：纯静态一次性报告(弃因：不符合讨债真实动态过程，落地率低)。
- 后果：贴近真实律师工作方式、随情报进化；代价是模块更多、需 compliance 严格把关施压合规性。
