# SPEC —— 个人合法讨债助手（V1，动态案件博弈）

> Phase 2 SPEC 产物（第16轮升级）。基于 DISCOVERY.md + 工厂已建能力 + 老板第16轮补充。
> ⚠️ 红线不变：仅合法路径；查财产=指引合法渠道/律师调查令；绝不施压催收/非法查信息。
> ⚠️ 核心成功标准（要点4）：实际讨回率，不是方案花哨；策略必须含"执行可行性"。

## 0. 核心理念升级（第16轮，老板补充点）—— 从"一次性报告"到"动态案件博弈"
老板指出：讨债是**持续动态博弈**，不是 LLM 出一份报告就完事。它要像一个真正的债务律师：
- **持续追踪债务人动态**：新情报随时进来（如"债务人当选村支书"→ 多了合法施压筹码：先告知利弊，不还则向其直属上级合理反映）；微信联系到本人得到的新信息也要纳入。
- **经常更新法务知识 + 了解当地执法情况**：法条/司法解释/本地执行口径会变。
- **每来一条新情报 → 重新评估策略**，而非静态报告。

→ 因此 V1 架构核心是 **"案件时间线 + 情报库 + 策略随情报动态重算"**（业界律所案件管理的标准做法：案件全周期 + 时间线动态跟踪 + 财产/情报线索动态清单）。
→ 合规护栏：所有"施压"建议必须是**合法**的（如向其任职单位/上级**依法依规反映**真实欠债事实属正当维权；但不得捏造、不得骚扰无关第三人、不得威胁恐吓——由 compliance 模块把关）。

## 1. 架构概述（V1，本地运行）

```
┌─────────────────────────────────────────────────────────────┐
│  debt-collection（本地 CLI/工具，纯本地存储，敏感数据不出本机）   │
│                                                              │
│  1) 债务台账(SQLite)  ← 录入7~8笔债务(债务人/金额/性质/时效/证据)  │
│  2) 资料整理  ← 复用工厂 ingestion-pipeline(合同PDF/借据图/录音)   │
│  3) 合规取数  ← 复用工厂 data-acquisition(L1清单+L2人在环)        │
│  4) 数据质量  ← sources.yaml(可信度分级) + 忠实度校验            │
│  5) 策略报告生成 ← 喂证据+取数结果+法条 → LLM 产出"讨债策略报告"   │
│  6) 时效预警  ← 按关键日期算诉讼时效，到点提醒                    │
└─────────────────────────────────────────────────────────────┘
        │ 模型走 LiteLLM 网关：策略推理优先 GLM，常规本地 qwen
        ▼
   本地模型(qwen3.6-35b) / GLM(cloud/glm-primary) / local/embedding(RAG)
```

复用工厂能力（不重造轮子）：
- **ingestion-pipeline**（FB-8）：把合同/借据/录音转结构化。
- **data-acquisition**（FB-6 L1/L2）：生成官方渠道待查清单 + 人在环查询 + 归档。
- **data-quality / sources.yaml**（FB-5）：来源可信度，防法律幻觉。
- **LiteLLM 网关 + local/embedding**：模型路由 + RAG。

## 2. 技术选型（候选对比）

| 维度 | 方案A（选定） | 方案B | 选择理由 |
|---|---|---|---|
| 形态 | 本地 CLI + SQLite | Web 应用 | A 最简、纯本地、隐私最佳；V1 不需要 Web |
| 存储 | SQLite（本地文件） | JSON 文件 | SQLite 支持查询/多笔管理，单文件易备份 |
| 语言 | Python 3.11 + uv | — | 与工厂一致，复用 Pattern |
| 策略推理 | GLM 优先，本地兜底 | 纯本地 | 法律推理质量 GLM 更稳（要点4落地优先）；本地省成本 |
| 法条防幻觉 | 法条来自国家法规库 + 忠实度校验 + 免责声明 | LLM 直出 | 防"编造法条"，这是法律项目命门 |
| 财产/执行信息 | 合规取数层(L1/L2) | 自动爬虫 | 合规、不封号（红线） |

## 3. 关键决策（ADR）

- **ADR-001**：本地 CLI + SQLite 形态（隐私优先，V1 不做 Web）。
- **ADR-002**：策略推理默认 GLM、本地兜底；法条强制来自法规库并做忠实度校验（防法律幻觉）。
- **ADR-003**：所有外部数据经"合规取数层"，绝不自动爬强风控/非法查信息。
- **ADR-004**：策略报告必须含"执行可行性 + 回款概率"段（落实要点4，反老赖执行难）。
- **ADR-005**：敏感数据（身份证号/借据）纯本地、脱敏存储，不出本机、不入 Git。

## 4. 核心 Feature 与验收（对应 DISCOVERY 的 AC）

| Feature | 说明 | 验收(AC) |
|---|---|---|
| F1 债务台账 | 录入/管理 ≥8 笔；字段：债务人/金额/性质/关键日期/证据清单/阶段 | AC-1 |
| F2 资料整理 | 调 ingestion 把合同/借据/录音转结构化并关联到某笔债务 | AC-2 辅助 |
| F3 时效预警 | 按关键日期算诉讼时效(一般3年)剩余/超期 | AC-3 |
| F4 合规取数 | 调 acquisition 生成某债务人待查清单 + 归档结果 | AC-4 |
| F5 策略报告 | 综合证据+取数+法条 → 输出策略报告(含执行可行性) | AC-2/AC-6 |
| F6 合规自检 | 报告内容扫描：无非法/施压性建议 | AC-4 |

## 5. 策略报告结构（F5 核心产出物，支持动态更新）
每笔债务生成/更新一份报告，固定包含：
1. **证据体检**：证据链是否够立案/胜诉（借条/转账/聊天记录齐不齐）。
2. **诉讼时效**：是否在 3 年内、是否需立即中断时效。
3. **执行可行性预判（要点4，最重要）**：先评"赢了能不能执行到财产"——对方有无失信记录/可执行财产线索/规避执行历史；若执行难，提示诉前财产保全。
4. **合法博弈筹码（第16轮新增）**：基于情报库识别的**合法**筹码及用法。
   - 例：债务人当选村支书 → ①先正式告知欠债不还的法律后果与对其信用/任职的潜在影响；②若仍不还，**依法依规**向其任职单位/上级反映**真实**欠债事实（这是正当维权，非骚扰）。
   - ⚠️ 每个筹码都过 compliance：不得捏造事实、不得威胁恐吓、不得骚扰无关第三人、不得公开侮辱。
5. **讨债阶梯**：协商 → 律师函 → 支付令 → 起诉 → 执行；每步成本/周期/回款概率。
6. **合法取数清单**：该去哪些官方渠道查（调 acquisition）。
7. **话术/要点**：合规协商话术、文书要点（不替代律师，标注"建议律师复核"）。
8. **本次更新说明（动态重算时）**：因哪条新情报触发了哪些策略调整。
9. **免责声明**：本报告为辅助参考，重大决策请咨询执业律师。

## 5.5 情报库与动态重算（第16轮新增，核心）
- **情报（Intel）**：每条新信息一条记录——来源(朋友/微信/官方查询/媒体)、内容、可信度(关联 sources.yaml)、时间、是否影响策略。
  - 例：`{来源:朋友, 内容:"债务人当选村支书", 可信度:中, 影响:新增合法施压筹码}`。
- **案件时间线（Timeline）**：把"债务关键日期 + 每次沟通 + 每条情报 + 每次取数"按时间串成一条线，一眼看清案件演进。
- **动态重算**：新增情报后，`strategy` 重新评估并生成**增量策略更新**（不是从头重写），标注"因 X 情报，建议新增 Y 手段"。
- **筹码识别**：从情报中识别**合法**博弈筹码（如任职单位可依法反映、有可执行财产线索、面子/信用顾虑），并评估每个筹码的合规性与有效性。

## 6. 模块划分（src/）
```
src/debt/
├── models.py        # 债务/证据/情报/时间线/报告数据模型 + SQLite
├── ledger.py        # 台账 CRUD（多笔债务）
├── intel.py         # 情报库：录入/管理新情报，标注来源可信度与策略影响
├── timeline.py      # ① 诉讼时效计算 ② 案件时间线聚合(债务+沟通+情报+取数)
├── strategy.py      # 策略生成 + 动态重算(新情报→增量更新)；合法筹码识别
├── knowledge.py     # 法务知识：从法规库取法条 + 本地执行口径，供策略引用(防幻觉)
├── compliance.py    # 合规自检：扫描非法/骚扰/威胁/捏造内容，确保施压建议合法
└── cli.py           # 命令行入口
tests/               # 时效计算/合规自检/情报影响判定/筹码合规性 可纯逻辑测
```

## 7. 风险（详见 RISK.md）
- 法律幻觉（最高）→ 法条来自法规库 + 忠实度校验 + 免责声明 + 建议律师复核。
- 执行难 → 策略前置执行可行性评估。
- 隐私泄露 → 纯本地、脱敏、不入 Git。
- 模型能力 → 策略推理用 GLM；本地兜底标注"待复核"。
