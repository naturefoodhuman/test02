# SPEC —— 个人合法讨债助手（V1）

> Phase 2 SPEC 产物。基于 DISCOVERY.md（含 6 点补充意见）+ 工厂已建能力。
> ⚠️ 红线不变：仅合法路径；查财产=指引合法渠道/律师调查令；绝不施压催收/非法查信息。
> ⚠️ 核心成功标准（要点4）：实际讨回率，不是方案花哨；策略必须含"执行可行性"。

## 1. 架构概述（V1，本地运行）

```
┌─────────────────────────────────────────────────────────────┐
│  debt-collection（本地 CLI/工具，纯本地存储，敏感数据不出本机）   │
│                                                              │
│  1) 债务台账(SQLite)  ← 录入7~8笔债务(债务人/金额/性质/时效/证据)  │
│  2) 资料整理  ← 复用工厂 ingestion-pipeline(合同PDF/借据图/录音)   │
│  3) 合规取数  ← 复用工厂 data-acquisition(L1清单+L2人在环)        │
│  4) 数据质量  ← sources.yaml(可信度分级) + 忠实度校验            │
│  5) 策略报告生成 ← 喂证据+取数结果+法条 → LLM 产出"讨债策略报告"   │
│  6) 时效预警  ← 按关键日期算诉讼时效，到点提醒                    │
└─────────────────────────────────────────────────────────────┘
        │ 模型走 LiteLLM 网关：策略推理优先 GLM，常规本地 qwen
        ▼
   本地模型(qwen3.6-35b) / GLM(cloud/glm-primary) / local/embedding(RAG)
```

复用工厂能力（不重造轮子）：
- **ingestion-pipeline**（FB-8）：把合同/借据/录音转结构化。
- **data-acquisition**（FB-6 L1/L2）：生成官方渠道待查清单 + 人在环查询 + 归档。
- **data-quality / sources.yaml**（FB-5）：来源可信度，防法律幻觉。
- **LiteLLM 网关 + local/embedding**：模型路由 + RAG。

## 2. 技术选型（候选对比）

| 维度 | 方案A（选定） | 方案B | 选择理由 |
|---|---|---|---|
| 形态 | 本地 CLI + SQLite | Web 应用 | A 最简、纯本地、隐私最佳；V1 不需要 Web |
| 存储 | SQLite（本地文件） | JSON 文件 | SQLite 支持查询/多笔管理，单文件易备份 |
| 语言 | Python 3.11 + uv | — | 与工厂一致，复用 Pattern |
| 策略推理 | GLM 优先，本地兜底 | 纯本地 | 法律推理质量 GLM 更稳（要点4落地优先）；本地省成本 |
| 法条防幻觉 | 法条来自国家法规库 + 忠实度校验 + 免责声明 | LLM 直出 | 防"编造法条"，这是法律项目命门 |
| 财产/执行信息 | 合规取数层(L1/L2) | 自动爬虫 | 合规、不封号（红线） |

## 3. 关键决策（ADR）

- **ADR-001**：本地 CLI + SQLite 形态（隐私优先，V1 不做 Web）。
- **ADR-002**：策略推理默认 GLM、本地兜底；法条强制来自法规库并做忠实度校验（防法律幻觉）。
- **ADR-003**：所有外部数据经"合规取数层"，绝不自动爬强风控/非法查信息。
- **ADR-004**：策略报告必须含"执行可行性 + 回款概率"段（落实要点4，反老赖执行难）。
- **ADR-005**：敏感数据（身份证号/借据）纯本地、脱敏存储，不出本机、不入 Git。

## 5. 策略报告结构（F5 核心产出物）
每笔债务生成一份报告，固定包含：
1. **证据体检**：证据链是否够立案/胜诉（借条/转账/聊天记录齐不齐）。
2. **诉讼时效**：是否在 3 年内、是否需立即中断时效。
3. **执行可行性预判（要点4，最重要）**：先评"赢了能不能执行到财产"——对方有无失信记录/可执行财产线索/规避执行历史；若执行难，提示诉前财产保全。
4. **讨债阶梯**：协商 → 律师函 → 支付令 → 起诉 → 执行；每步成本/周期/回款概率。
5. **合法取数清单**：该去哪些官方渠道查（调 acquisition）。
6. **话术/要点**：合规协商话术、文书要点（不替代律师，标注"建议律师复核"）。
7. **免责声明**：本报告为辅助参考，重大决策请咨询执业律师。

- 隐私泄露 → 纯本地、脱敏、不入 Git。
- 模型能力 → 策略推理用 GLM；本地兜底标注"待复核"。
