# HARDEN 检查清单 —— debt-collection（Phase 4 加固）

> HARDEN 阶段目标：交付前的安全/合规/健壮性加固。对个人讨债项目，**隐私 + 法律合规**是重中之重。
> 检查方式：自动(脚本/测试) + 人工(GLM 安全审查，降级则本地+标注待复核)。

## 一、隐私与数据安全（ADR-005，最高优先级）
- [ ] **敏感数据不入库**：`runtime/`、`*.db`、`data/` 已在 .gitignore；`git status` 确认数据库不被跟踪。
- [ ] **身份证脱敏**：展示处用 `mask_id()`（前3后4打码）；DB 里如存明文需评估是否加密。
- [ ] **数据不外传**：策略推理走本地 LiteLLM 网关（GLM 经 ModelScope）——⚠️ 注意：案件事实会发给 GLM 云端。
      → 决策点：敏感案件是否改用**本地模型**(local/primary)推理以完全不出本机？(见下"待老板决断")
- [ ] **录音/借据等原始资料**：经 ingestion 后存本地；不上传。
- [ ] **临时文件清理**：诊断/取数产生的临时文件不残留敏感信息。

## 二、法律合规红线（项目命门）
- [ ] **合规自检覆盖**：compliance 模块拦截 8 类入刑催收行为 + 非法查信息 + 骚扰第三人。
- [ ] **策略报告必过 compliance**：strategy 生成后强制 check_text；未通过给出违规项。
- [ ] **法律幻觉防护**：法条来自 knowledge(回溯法规库)；报告含免责声明+"建议律师复核"。
- [ ] **施压建议合法性**：向单位/上级反映=正当维权(放行)，捏造/威胁/骚扰(拦截)——抽样人工复核。
- [ ] **取数合规**：只走 L1/L2，不自动爬强风控/非法查；社交情报经数据质量分级。

## 三、代码健壮性
- [ ] **输入校验**：金额非负、日期格式、必填字段缺失的处理。
- [ ] **异常处理**：DB 操作、LLM 调用、文件读写有 try/except，不裸崩。
- [ ] **降级路径**：无 GLM 网关→离线模板；无工厂能力→提示；缺关键日期→时效返回 unknown。
- [ ] **测试覆盖**：核心模块(时效/合规/台账/策略/CLI)有单测；当前 18 passed。
- [ ] **依赖安全**：核心零三方依赖；可选依赖(litellm/工厂能力)缺失不影响核心。

## 四、性能与边界
- [ ] **多笔规模**：≥8 笔台账 OK(AC-1 已测)；更多笔(50+)性能可接受。
- [ ] **长录音**：FunASR 转写长音频较慢→属预期(隐私优先)，给进度提示。
- [ ] **大 PDF**：MinerU 慢→属预期；超时有保护(CLI timeout 1800s)。

## 五、自动安全自检（脚本）
- [ ] 运行 `bash docs/harden/security_scan.sh`（见同目录），检查：.gitignore 覆盖、是否误提交 .db、compliance 关键词覆盖。

## 老板决断（第18轮已定）
1. **策略推理模型 = 每次可选（决策C）**：`debt report --model cloud/glm-primary`(质量,出云端) 或 `--model local/primary`(隐私,不出本机)。
   - 隐私模式会提示"案件事实不出本机"；质量模式会提示"会发送云端"。已实现并测试。
2. **DB 加密 = 以后再说（决策C）**：V1 明文 SQLite + 本地 + .gitignore，个人自用足够；后续需要再加 SQLCipher。

## 退出门控
- 自动：security_scan 通过 + 18 测试通过。
- 人工：HITL Gate-4（最终验收，老板判断质量与合规）。
